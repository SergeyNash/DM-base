## UX-принципы DM-base

1. **AppSec-first сценарии.** Основная пользовательская задача — быстро загрузить SARIF, отфильтровать критичные findings и поделиться контекстом. Все UX-решения проверяем на вклад в этот flow.
2. **Zero learning curve.** Первый экран — dropzone + последние отчёты; никаких дополнительных настроек до первого успешного просмотра.
3. **Слои информации.** Используем прогрессивное раскрытие: high-level summary (кол-во findings, тренды), затем детали rule/run, затем исходный SARIF JSON.
4. **Стабильные фильтры.** Фильтры и сортировки живут в URL/hash + localStorage, чтобы при шаринге ссылки коллега видел тот же контекст.
5. **Сигналы доверия.** Всегда показываем источник данных (локальный файл vs Supabase), время последней синхронизации и статус обработчика Netlify Function.
6. **Accessibility и скорость.** Поддерживаем клавиатурную навигацию, high-contrast тему, skeleton состояния и lazy rendering больших таблиц.
7. **Ошибки в понятных терминах.** Сообщения объясняют, что произошло (например, «Supabase недоступен») и предлагают actionable шаг (повторить, скачать лог, открыть issue).
8. **Простой визуальный язык.** Используем предсказуемую сетку, одноуровневую навигацию и понятные подписи (по возможности — термины OWASP/AppSec), избегаем скрытых действий и перегруженных таблиц.
9. **Фиксированная цветовая палитра.** Базовые цвета: фон #f5f7fb, текст #0f172a, акцентная плашка #0f172a/#ffffff, вторичные CTA и состояния — оттенки индиго (#a0b8ff, #d0defa) и статусные цвета (#fee4e2 для ошибок, #eef2ff для инфо). Новые компоненты наследуют эти токены через CSS custom properties.
10. **Единый стиль компонентов.** Скругления 20–24px, тень `rgba(15, 23, 42, 0.06)`, вертикальный ритм 1.5–2rem; все интерактивные элементы имеют hover/focus состояния и размеры не менее 40px по каждому измерению.
