## Принципы разработки DM-base

1. **Security-by-default.** Любая новая функциональность проверяется на влияние на конфиденциальность SARIF-данных и принципы минимизации прав (Supabase RLS, ограничение session scope).
2. **Фичи через пользовательские задачи.** Каждую итерацию начинаем с конкретных AppSec pain points (загрузка отчёта, triage, сравнение версий) и только потом выбираем технологии.
3. **Модульность + переиспользование.** Общая логика (нормализация SARIF, работа с сессиями) размещается в `src/lib` и импортируется и в Netlify functions, и во фронтенд.
4. **Чистые pull-request’ы.** Один PR = одна функциональность; обязателен чек‑лист: lint, unit-тесты (`npm test`), e2e smoke (если затронут UI), обновление документации.
5. **Прозрачные окружения.** Любое отличие между локальной разработкой, Netlify Preview и Production фиксируем в `docs/environment-diffs.md` (завести при необходимости).
6. **Observability.** Добавляем минимум telemetry hooks (Supabase logs, Netlify function logs) для каждой критичной ветки пользовательского пути.
7. **Автоматизация прежде ручных шагов.** Скрипты в npm для типовых задач (линт, формат, миграции), чтобы исключить дрейф инструкций.
